## 21. ВАЖНО: SSOT И ПИЛОТНЫЙ РЕЖИМ

### Проблема:
Раздел Новости использует множество SSOT компонентов, которые шарятся с другими разделами:
- `unified_card.html` → Галерея, GrowReports, Новости
- `unified_grid.css` → сетка карточек везде
- `base_list_page.html` → все списки
- Header, Footer, Sidebar → глобальные

**Исключение:** Магазин — другая карточка (товарная)

### Стратегия пилотного режима:

**Вариант A: Изоляция через классы**
```html
<!-- Новый дизайн только для news -->
<div class="news-redesign">
  <!-- новые стили применяются только внутри -->
</div>
```

**Вариант B: Отдельные шаблоны**
```
templates/news/
├── news_list_v2.html      # Новый шаблон

## 1. ПРОБЛЕМЫ ТЕКУЩЕГО ДИЗАЙНА

### 1.1 Выявленные проблемы
- Карточки слишком большие на мобильном (занимают весь экран)
- Половина карточки - метаданные, половина - превью (громоздко)
- На FullHD не занимает всю ширину страницы
- Дизайн выглядит "несовременно и временно"
- Слишком много информации на карточке (3 счётчика, автор, дата, описание)

### 1.2 Пожелания пользователя
- Узкие горизонтальные прямоугольники для мобильного
- Переключатель видов (сетка / список)
- Показывать прочитанные/непрочитанные новости
- Badge "Оригинальный контент" со значком
- Красивая подложка/фон ("вата")
- Lightbox для изображений

В прототипе v1 заложена адаптивная сетка:
```
grid-cols-1      → мобильный (1 карточка)
sm:grid-cols-2   → планшет (2 карточки)
lg:grid-cols-3   → ноутбук (3 карточки)
xl:grid-cols-4   → FullHD (4 карточки)
