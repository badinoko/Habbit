# Промпт ГЛУБОКОГО ИССЛЕДОВАНИЯ референсного сайта GrowDiaries (v2)
> Этот документ — только для **съёма фактов** (growdiaries.com). Все результаты складываются в `docs/GROWREPORTS_REFERENCE_SPECS/…`, индекс — в `docs/GROWREPORTS_REFERENCE_AUDIT.md`. Любое утверждение должно подтверждаться артефактом (скриншот/DOM/метрика/лог теста).

## Цель

Получить исчерпывающее, воспроизводимое описание каждого ключевого экрана и UI‑элемента `growdiaries.com`, достаточное для «слепого» повторения в нашем проекте. Результат — единый SSOT‑набор артефактов: скриншоты, DOM‑карты, размеры/отступы/сетки, токены стиля, поведение и сценарии, A11y/CLS/перфоманс.

## Инструменты (используются совместно)

- Playwright — полноэкранные скриншоты и сценарии (desktop/mobile)
- Selenium — при сложных интеракциях/негативных сценариях (по необходимости)
- Content7 — извлечение структуры/текстов/семантики (по необходимости)
- DevTools — Layout/Box Model/Computed, Performance, Lighthouse, Network, Accessibility/ARIA
- Axe DevTools — A11y‑проверки
- CDP (Chrome DevTools Protocol) — съём из авторизованного Chrome (`http://127.0.0.1:9222`) для обхода антибот‑ограничений
- Антибот‑настройки — подмена UA, скрытие `navigator.webdriver`, отсечение трекеров (реализовано в спеках/скриптах)

## Артефакты и структура вывода

- `docs/GROWREPORTS_REFERENCE_AUDIT.md` — главный индекс (ссылки на артефакты)
- `docs/GROWREPORTS_REFERENCE_SPECS/<page>/`:
  - `README.md` — назначение, сценарии
  - `screens/<breakpoint>.png` — скриншоты 320/375/768/992/1200/1440/1920
  - `dom/<section>.json` — DOM‑карта: иерархия, селекторы, роли/ARIA
  - `metrics/<breakpoint>.json` — размеры/отступы/сетка/типографика/цвета/z‑index/тени
  - `flows/<flow>.md` — сценарии взаимодействия (скролл, фильтры, lightbox и т.д.)
  - `perf/<breakpoint>.json` — LCP/CLS/INP/бандлы/шрифты/lazy‑загрузка
  - `a11y/<breakpoint>.json` — отчёт axe, роли/ARIA, таб‑порядок, контрасты
- `docs/GROWREPORTS_COMPONENT_SPECS.md` — регистр компонентов (карточки, бейджи, фильтры, таймлайн, правое меню, мини‑чарты, формы недели, лайтбокс и т.д.)
- `docs/GROWREPORTS_TOKENS.md` — таблица токенов (цвета, размеры, сетка, шрифты, тени) + маппинг в наши переменные

## Обязательные страницы/блоки (P0)

- Список дневников (аналог `/growreports/`): сетка карточек, фильтры/сортировки/чипы, пагинация/ленивая загрузка
- Деталь дневника (таймлайн недель): Hero + equipment bar + «Week N» (фото/метрики/заметки), правое sticky‑меню, мобильный «Недели ▾», лайтбокс
- Профиль автора → список его дневников
- Создание/редактирование дневника (wizard/stepper)
- Редактирование недели (форма/модал)
- Карточка дневника (в списке): обложка, бейджи стадии/Wn/«NEW», мета, счётчики
- Лайтбокс недели: навигация, подписи, группы
- Поиск/фильтры: стадии/типы/среды/сортировки/хэштеги

## Шаблон описания компонента

- Идентификатор; назначение
- Визуал (скрины @2x), размеры, отступы, сетка (колонки/gutters), брейкпоинты
- Типографика (семейство/кегль/межстрочие/вес)
- Цвета/градиенты/тени/радиусы/иконки (HEX/RGBA/веса)
- Состояния (hover/focus/active/disabled/sticky/empty/loading/error)
- Локализуемые тексты/лейблы/хинты
- Доступность (роль, aria, таб‑порядок, фокус‑видимость, контраст)
- Логика/сценарии (клик/скролл/моб.‑поведение/пустые данные)
- Зависимости (токены/переменные/шрифты)
- Маппинг на наши токены/переменные

## Методика съёма данных (строго)

1. Скриншоты: полноэкранные на брейкпоинтах 320/375/768/992/1200/1440/1920 + фрагменты (карточка, hero, фильтры, неделя, правое меню, модалки, формы)
2. DOM‑карта: семантическая структура, ключевые селекторы, роли/ARIA, landmarks, таб‑порядок
3. Измерения: DevTools → Box Model/Computed — размеры, paddings/margins, font, цвет; сетка: ширина контейнера, колонки, gutter, breakpoints
4. Поведение: сценарии (фильтры, scrollspy, раскрытие недель, лайтбокс, редактирование); мобильные («Недели ▾», свайпы, sticky)
5. Перфоманс: LCP/CLS/INP по брейкпоинтам; lazy‑load; критический CSS; шрифты (preload); бандлы
6. A11y: axe‑отчёты, роли/aria, фокус‑кольца, alt‑тексты, контрасты (WCAG AA)
7. Токены: палитра, типографика, радиусы, тени, интервалы сетки → маппинг в наши переменные

## Сценарии Playwright/CDP (минимум)

- `growdiaries_reference_list.spec.ts`: скриншоты/DOM/метрики списка (сетка/карточки/фильтры/пагинация)
- `growdiaries_reference_detail.spec.ts`: деталь (hero/equipment/недели/правое меню/мобильный/лайтбокс) с fallback‑навигациями
- `growdiaries_reference_author.spec.ts`: профиль автора → список дневников
- `growdiaries_auth.spec.ts`: логин через `/auth/signin` (ручная капча)
- CDP‑скрипты: `cdp_capture_detail.js`, `cdp_capture_author.js` — съём из авторизованного Chrome (screens/dom/metrics/perf/a11y)

## Фиксация фактов

- Только наблюдения и числа; у каждого утверждения — источник (скрин/DOM/метрика)
- Любые выводы без артефакта — не принимаются
- В `docs/GROWREPORTS_COMPONENT_SPECS.md` переносить номера/размеры со ссылками на соответствующие `metrics/*.json` и скриншоты

## Acceptance Criteria исследования

- По каждой странице — полный комплект артефактов (screens/dom/metrics/flows/perf/a11y)
- По каждому ключевому компоненту — спецификация по шаблону с ссылками на артефакты
- Таблица токенов и маппинг в наши переменные готовы
- Есть «реализационная матрица»: референс → наш шаблон/файл/компонент (с оценкой трудозатрат)

## Команды (наше окружение)

- Перезапуск/пинг (при необходимости локального стенда): `pwsh -NoProfile -File scripts/besedka_ops.ps1 -Action restart_django_full -WaitSeconds 40`
- Прогон E2E референса: `npx --yes playwright test tests/playwright/growdiaries_reference_*.spec.ts --reporter line`
- CDP‑съём (авторизованный Chrome на `127.0.0.1:9222`): запуск соответствующих `tests/playwright/cdp_capture_*.js`
