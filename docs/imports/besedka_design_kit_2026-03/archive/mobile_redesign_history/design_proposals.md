# Mobile Adaptation - Design Proposals

> **SSOT:** Эскизы и варианты дизайна для мобильной адаптации
> **Создан:** 2026-02-20
> **Статус:** ОЖИДАЕТ УТВЕРЖДЕНИЯ

---

## 📱 Общая философия мобильной адаптации

**Принципы:**
- Контент важнее декора
- Минимум пустого пространства
- Одинаковые паттерны во всех разделах
- Очевидная разница между grid/list режимами

**Референс:** News раздел (лучшая реализация на данный момент)

---

## 🎯 ПРОБЛЕМА #1: Gallery List View - превью "каменное"

### Текущая реализация (ЧТО НЕ ТАК):

```
┌──────────────────────────────────────────┐
│ ┌────────────────────────────────────┐   │
│ │                                    │   │
│ │        ОГРОМНОЕ ИЗОБРАЖЕНИЕ        │   │ ← 200-300px высота
│ │         (занимает 80% карточки)    │   │   Почти как в grid view
│ │                                    │   │
│ └────────────────────────────────────┘   │
│                                          │
│ Название фото обрезается...              │ ← 20% карточки
│ 👁️ 123  💬 5                             │   Мало метаданных
└──────────────────────────────────────────┘
```

**Проблема:**
- В list view превью такое же большое как в grid view
- Нет смысла переключаться между видами
- Название обрезается
- Мало места для метаданных

---

### ВАРИАНТ A: Маленькое квадратное превью (РЕКОМЕНДУЮ)

```
┌──────────────────────────────────────────┐
│ ┌─────┐ Полное название фото без         │
│ │     │ обрезания, может занимать        │
│ │80x80│ несколько строк                  │
│ │THUMB│                                  │ ← Компактно
│ └─────┘ Описание: Lorem ipsum dolor...   │   + много инфо
│         Автор: @username                 │
│         📅 12 янв 2026 · 👁️ 123 💬 5     │
└──────────────────────────────────────────┘
```

**Преимущества:**
- Очевидная разница с grid view
- Много места для текста
- Похоже на News list view (консистентность)
- Быстрее загружается

**Недостатки:**
- Меньше фокуса на изображении
- Может быть непривычно для галереи

---

### ВАРИАНТ B: Широкое горизонтальное превью

```
┌──────────────────────────────────────────┐
│ ┌──────────────────────┐                 │
│ │  WIDE THUMBNAIL      │ Название фото   │ ← 120x80px
│ │    (3:2 ratio)       │ Автор: @user    │   Компромисс
│ └──────────────────────┘ 👁️ 123  💬 5     │
└──────────────────────────────────────────┘
```

**Преимущества:**
- Больше фокуса на изображении чем в варианте A
- Всё равно компактнее чем grid view

**Недостатки:**
- Меньше места для текста
- Не так очевидна разница с grid view

---

### ВАРИАНТ C: Оставить как есть, но больше метаданных

```
┌──────────────────────────────────────────┐
│ ┌────────────────────────────────────┐   │
│ │     БОЛЬШОЕ ИЗОБРАЖЕНИЕ            │   │ ← Без изменений
│ └────────────────────────────────────┘   │
│                                          │
│ 📝 Полное название фото                  │ ← Добавить:
│ 📄 Описание: Lorem ipsum...              │   - Описание
│ 👤 Автор: @username                      │   - Полное имя автора
│ 📅 12 янв 2026 · 👁️ 123 💬 5             │   - Дату
│ 🏷️ Теги: #природа #закат                 │   - Теги
└──────────────────────────────────────────┘
```

**Преимущества:**
- Сохраняем фокус на изображении
- Добавляем больше полезной информации

**Недостатки:**
- Карточки станут очень высокими
- Всё равно не очень очевидна разница с grid view

**GrowReports (унифицированная версия):**

- Все фильтры/кнопки в ОДНУ строку
- Максимальная компактность
- Одинаковый паттерн везде

## 📋 ЗАДАЧИ ДЛЯ РЕАЛИЗАЦИИ (готово к переносу в overview)

### #MOBILE-101: Navbar - заголовок раздела в центре

### #MOBILE-102: Sticky bar - одна линия под navbar

- Sticky при скролле

### #MOBILE-103: Drawer - скролл + sticky footer
- overflow-y: auto для контента
- Кнопки Сбросить/Применить sticky внизу
- Glassmorphism blur для footer
