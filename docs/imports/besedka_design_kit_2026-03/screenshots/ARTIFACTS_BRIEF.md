# Бриф для Artifacts: Редизайн публичных страниц Беседки

> Этот документ — готовый промпт. Скопируй его в Artifacts и приложи скриншоты.

## Контекст проекта

**Беседка** — community-платформа (Django). Три основных раздела: Новости, Галерея, Гроурепорты. Плюс главная страница (лендинг). Стек: Django 4.2, чистый HTML/CSS/JS (без React, без Tailwind).

---

## Архитектура фронтенда (ОБЯЗАТЕЛЬНО учитывать)

### Фиксированные элементы (всегда на экране)

1. **Навбар** (top: 0, height: 64px, z-index: 1000)
   - Фиксирован вверху экрана
   - Стеклянный фон (glassmorphism: backdrop-filter blur 20px)
   - Содержит: гамбургер (мобилка) | логотип | навигация по разделам (десктоп) | уведомления | аватар пользователя
   - На мобилке: навигация по разделам скрывается, остается гамбургер + логотип + иконки справа

2. **Sticky Bar** (top: 64px, min-height: 63px, z-index: 150)
   - Прилипает под навбаром
   - Стеклянный фон (glassmorphism)
   - Содержит элементы управления текущего раздела:
     - Кнопка "Назад"
     - Переключатели (Фото/Альбомы в галерее; Все/Популярные/Обсуждаемые фильтры)
     - Кнопки действий (Добавить, Поиск, Фильтры, Переключение вида сетка/список)
   - На мобилке: переносится на 2 строки (верх: назад + переключатели; низ: кнопки действий)

3. **Боковая панель** (sidebar, гамбургер на мобилке)
   - Выезжает по клику на гамбургер
   - Содержит навигацию по разделам (Новости, Магазин, Гроурепорты, Галерея, Чат)

### Breakpoints

Используй только ДВА основных:
- **Desktop**: >= 768px (Full HD 1920x1080 — основной десктоп)
- **Mobile**: < 768px (375px — базовый мобильный, диапазон 320-430px)

Внутри мобильного диапазона вся вёрстка должна быть РЕЗИНОВОЙ (проценты, flex, не фиксированные пиксели), чтобы 320px и 430px выглядели одинаково без отдельных правил.

### Дизайн-система (CSS переменные)

```css
/* Цвета */
--bs-primary: #2EA6FF;           /* Основной синий */
--bs-body-bg: #ffffff;           /* Фон */
--bs-text-primary: #000000;      /* Основной текст */
--bs-text-secondary: #6c757d;    /* Второстепенный текст */
--bs-surface-1, --bs-surface-2;  /* Поверхности карточек */
--bs-card-bg;                    /* Фон карточки */

/* Карточки */
--card-border-radius: 20px;
--card-shadow: 0 8px 32px rgba(0,0,0,0.12);
--card-shadow-hover: 0 12px 48px rgba(0,0,0,0.18);

/* Стекло */
--glass-navbar: rgba(255,255,255,0.85);
--bs-backdrop-blur: blur(20px);

/* Тени */
--bs-shadow-sm, --bs-shadow, --bs-shadow-lg;
```

### Темы
Поддерживаются Light и Dark. Рисовать для Light, но учитывать что цвета — из переменных.

## Страницы для редизайна

### 1. Главная страница (лендинг)

**Текущее состояние:** Блок объявлений + прогресс-бар платформы + 3 колонки (Последние новости / Гроурепорты / Свежие фото). Выглядит как dashboard, не как лендинг.

**Что хочется:**
- Красивый, современный лендинг
- Hero-секция с приветствием
- Секции по разделам с превью контента (карточки последних новостей, фото, репортов)
- Чистый, воздушный дизайн
- Прогресс платформы можно оставить, но стилизовать

### 2. Листинг новостей / Листинг фото / Листинг гроурепортов

**Текущее состояние:** Сетка карточек (1-4 колонки в зависимости от ширины). Одинаковая unified карточка во всех разделах.

**Структура карточки (unified):**
- Изображение (aspect-ratio, обрезается)
- Бейджи поверх (категория, статус)
- Заголовок
- Описание (1-2 строки)
- Мета-инфо внизу: автор | дата | просмотры | комментарии

**Что хочется:**
- Более компактные и современные карточки (не такие высокие)
- Лучшая сетка (может, masonry? или staggered grid?)
- Больше воздуха между карточками
- На мобилке: 1 колонка, карточки на всю ширину
- На десктопе: 3-4 колонки

### 3. Детальная страница новости / фото

**Текущее состояние:** Двухколоночный layout:
- Левая (широкая): контент (фото или текст новости)
- Правая (sidebar, sticky): автор + кнопки + похожие

**Что хочется:**
- Чистый, читабельный layout
- На мобилке: sidebar скрывается, контент на всю ширину
- Блок автора — компактный
- Блок "Похожие" — горизонтальный скролл на мобилке

### 4. Детальная страница гроурепорта

**Текущее состояние:** Hero-блок (название, автор, мета-инфо, прогресс-бар) + информационная панель (сорт, оборудование) + записи по неделям.

**Что хочется:**
- Hero можно оставить, стилизовать
- Таймлайн записей (по неделям/дням) — визуально привлекательный
- Каждая запись: текст + фото + метрики + реакции + комментарии
- На мобилке: всё в одну колонку

## Технические требования к результату

### ВАРИАНТ A: Чистый HTML + CSS (предпочтительный)

```
Создай artifact в формате HTML. Используй ТОЛЬКО:
- HTML5
- CSS (custom properties / CSS variables)
- Никакого React, никакого Tailwind, никакого JSX
- Стили через <style> внутри artifact
- CSS Grid и Flexbox для layout
- Переменные из дизайн-системы выше
```

### ВАРИАНТ B: Если всё же TSX

```
Создай artifact в React, но:
- Все стили через inline CSS объект (style={{...}})
- НЕ использовать Tailwind классы
- Все цвета — как CSS var() или хардкод hex из дизайн-системы выше
- Комментарии о том, какой Django-блок это заменяет
```

## Порядок работы (рекомендация)

1. Начать с **листинга** (карточки) — он общий для всех трёх разделов
2. Затем **детальная страница** (фото/новость) — тоже общий layout
3. Затем **главная страница** (лендинг)
4. В конце **детальная гроурепорта** (уникальная)

Для каждой страницы делать и десктоп и мобильную версию в одном artifact.
