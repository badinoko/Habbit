#### 🐛 КРИТИЧЕСКИЙ БАГ: Обнуление прогресса при сохранении

**Проблема обнаружена пользователем:**
> "Я нажимаю кнопку «Обновить прогресс», и получается 45%. Я нажимаю кнопку «Сохранить изменения», и всё обновляется, кроме страницы главная. Ну, кроме компонента главная. Там остаётся 56%. Это очень странно."

**Анализ проблемы:**
1. Текст "Сейчас работаю над" исчез после удаления дублирующего объекта CurrentTasks #2
2. При клике на "Сохранить изменения" все прогресс-поля обнулялись (87/100/59/29/56/23 → 0/0/0/0/0/0)
3. Главная страница показывала старые значения из кэша

**Корневая причина:**
- Прогресс-поля были сделаны `readonly` (нередактируемые)
- **Readonly поля НЕ отправляются в POST запросе**
- View функция `current_tasks_view()` парсила POST для обновления полей
- Не находила прогресс-поля в POST → использовала значение по умолчанию (пустая строка)
- Функция `parse_progress('')` возвращала 0
- Сохранение: все прогресс-поля обнулялись в БД
