# Besedka Breakpoints (SSOT)

> **Единый источник правды** для всех модулей проекта.
> Последнее обновление: 2026-02-18

---

## Основные breakpoints

| Название | CSS px | Устройство | Примечание |
|:---------|:-------|:-----------|:-----------|
| **Mobile S** | 320px | Старые/маленькие телефоны | Минимальная поддерживаемая ширина |
| **Mobile M** | 412px | Xiaomi 12 Pro | Основной тестовый смартфон |
| **Mobile L** | 430px | iPhone 14 Pro Max | Крупные флагманы |
| **Tablet** | 768px | Планшеты, iPad | Переход на 2 колонки |
| **Desktop S** | 1024px | Маленькие ноутбуки | Опционально |
| **Desktop** | 1920px | Full HD | Основной рабочий режим |

---

## Коридор смартфонов (2024-2026)

Большинство современных смартфонов попадают в диапазон **360px — 430px** (CSS pixels).

| Диапазон | Что попадает |
|:---------|:-------------|
| 360-390px | Samsung Galaxy S series, Pixel |
| 390-414px | iPhone 12/13/14/15 standard |
| 412-430px | Xiaomi, OnePlus, iPhone Pro Max |

**Вывод:** Если работает на 320px и 430px — работает везде.

---

## Подход: Mobile-First

```css
/* База = телефон (320-430px) */
.component {
  /* мобильные стили */
}

/* Tablet (768px+) */
@media (min-width: 768px) {
  .component {
    /* планшетные стили */
  }
}

/* Desktop (1200px+) */
@media (min-width: 1200px) {
  .component {
    /* десктопные стили */
  }
}
```

## Практические правила

### 1. Сетки (Grid)

```css
/* Карточки автоматически адаптируются */
.bs-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(300px, 1fr));
  gap: 1rem;
}
```

### 2. Фото-превью (из GrowReports)

| Breakpoint | Поведение |
|:-----------|:----------|
| >= 768px | 1 ряд, 10 превью |
| < 768px | 2 ряда по 5 превью |

### 3. Комментарии (вложенные)

| Breakpoint | Поведение |
|:-----------|:----------|
| >= 768px | Полные отступы вложенности |
| < 768px | Сжатые отступы, узкие направляющие |

### 4. Toolbar / Sticky bar

| Breakpoint | Поведение |
|:-----------|:----------|
| >= 576px | Текст + иконки |
| < 576px | Только иконки, текст скрыт |

## Тестовая матрица

При проверке адаптивности использовать эти ширины:

```
320, 412, 430, 768, 1024, 1920
```

Минимум — **320, 430, 768, 1920** (4 точки).
