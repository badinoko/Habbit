# API-контракт (Web-first)

## 1. Назначение документа
`api_contract.md` фиксирует контракт HTTP-интерфейсов HabitFlow с учетом текущей архитектуры проекта.

Ключевой принцип:
- основной интерфейс проекта: серверные веб-роуты (`HTML + Redirect + AJAX JSON`);
- дополнительный интерфейс (опционально): отдельный JSON API под префиксом `/api`.

## 2. Позиционирование
Этот контракт:
- не превращает проект в API-first;
- не требует обязательной реализации полного REST API для дипломного минимума;
- описывает в первую очередь фактические текущие роуты и правила их эволюции.

## 3. Общие соглашения
- Все даты/время в JSON-ответах: ISO 8601 в UTC.
- Для HTML-форм используется `POST` + `303 See Other` после успешного создания.
- AJAX-операции (через `fetch`) должны возвращать JSON-консистентно в success/error ветках.
- Идемпотентные удаления допустимы (`204` даже если ресурс уже отсутствует), если это явно зафиксировано в контракте.

## 4. Основной контракт: текущие веб-роуты (обязательная часть)

### 4.1 Базовые страницы
| Метод | Путь | Назначение | Успех | Формат |
|---|---|---|---|---|
| `GET` | `/` | Главная страница | `200` | `text/html` |
| `GET` | `/soon` | Заглушка раздела в разработке | `200` | `text/html` |

### 4.2 Themes
| Метод | Путь | Назначение | Успех | Формат |
|---|---|---|---|---|
| `GET` | `/themes/` | Список тем | `200` | `text/html` |
| `GET` | `/themes/new` | Форма создания темы | `200` | `text/html` |
| `POST` | `/themes/` | Создание темы (форма) | `303` -> `/themes` | redirect |
| `GET` | `/themes/{name}` | Страница редактирования темы | `200` | `text/html` |
| `PUT` | `/themes/{name}` | Обновление темы (AJAX) | `200` | `application/json` |
| `DELETE` | `/themes/{name}` | Удаление темы (AJAX) | `204` | empty |

Ошибки и детали:
- `POST /themes/`: `400/500` с HTML-страницей ошибки.
- `GET /themes/{name}`: при отсутствии темы `404`.
- `PUT /themes/{name}`: `404/500` в JSON.

### 4.3 Tasks
| Метод | Путь | Назначение | Успех | Формат |
|---|---|---|---|---|
| `GET` | `/tasks/` | Список задач | `200` | `text/html` |
| `GET` | `/tasks/new` | Форма создания задачи | `200` | `text/html` |
| `GET` | `/tasks/{task_id}` | Страница редактирования задачи | `200` | `text/html` |
| `POST` | `/tasks/` | Создание задачи (форма) | `303` -> `/tasks` | redirect |
| `PUT` | `/tasks/{id}/edit` | Обновление задачи (AJAX) | `200` | `application/json` |
| `PATCH` | `/tasks/{id}/complete` | Отметить выполненной | `200` | `application/json` |
| `PATCH` | `/tasks/{id}/incomplete` | Снять выполнение | `200` | `application/json` |
| `DELETE` | `/tasks/{task_id}` | Удаление задачи (AJAX) | `204` | empty |

Ошибки и детали:
- `POST /tasks/`: `422` (валидация формы FastAPI), `400/500` (HTML-ошибка).
- `PATCH /tasks/{id}/complete|incomplete`: `404/500` в JSON.
- `DELETE /tasks/{task_id}`: `500` при внутренней ошибке.

## 6. Дополнительный (опциональный) JSON API `/api/*`
Этот раздел является дополнительным треком, а не обязательным минимумом.

Планируемые домены:
1. `Auth v1` (`/api/auth/*`)
2. `Habits v1` (`/api/habits/*`)
3. `Statistics v1` (`/api/statistics/*`)

Важно:
- реализация этого слоя возможна после закрытия обязательного web-контракта;
- при реализации `/api/*` он не должен ломать существующие веб-сценарии.

## 7. Матрица доступа (текущее состояние и целевое)
### 7.1 Текущее состояние
- Роуты `tasks/themes` доступны без обязательной авторизации.

### 7.2 Целевое состояние после `Auth v1`
- Все state-changing операции (`POST/PUT/PATCH/DELETE`) требуют авторизации.
- Операции чтения остаются публичными или защищаются по итоговому решению по `Auth v1`.

## 8. Критерии приемки этого контракта
Контракт принят, когда:
1. Все текущие роуты (`/`, `/soon`, `/tasks/*`, `/themes/*`) описаны и согласованы с фактическим поведением.
2. Нормализации `N1..N3` внедрены в код и покрыты тестами.
3. `backend_roadmap.md` и тесты не противоречат этому документу.
4. Опциональный слой `/api/*` не считается блокером дипломного минимума.
