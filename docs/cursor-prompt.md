# Промпт имплементации редизайна HabitFlow

## Контекст

Ты работаешь над проектом HabitFlow — персональным веб-приложением на FastAPI + Jinja2 + PostgreSQL.
Каноническая папка: `C:\Users\user\Projects\HabitFlow-git`
Шаблоны: `src/templates/` (Jinja2, `.html`)
Статика: `src/static/` (CSS и JS файлы)
Стек фронтенда: **чистый HTML + CSS + vanilla JS**, никакого React, TSX, Vue или сборщиков.

## Задача

Реализовать полный визуальный редизайн фронтенда по готовым HTML-артефактам (прототипам).
Артефакты — это standalone-файлы с встроенным CSS и JS, отрисованные в Claude Artifacts.
Твоя задача: перенести дизайн в реальные Jinja2-шаблоны проекта.

## Дизайн-система (извлечь в общий файл)

Создай файл `src/static/css/design-system.css` со следующими CSS-переменными и базовыми компонентами.
Этот файл подключается во всех шаблонах через `base.html`.

### CSS-переменные

```css
:root {
  --navy:    #1A1F36;
  --accent:  #6C63FF;
  --accent2: #8B84FF;
  --mint:    #00C9A7;
  --danger:  #FF6B6B;
  --warn:    #FFB347;
  --bg:      #F4F5FA;
  --surface: #FFFFFF;
  --border:  #E8EAF0;
  --text:    #1A1F36;
  --muted:   #8A90A8;
  --nav-h:   58px;

  /* цвета тем пользователя */
  --c-work:   #3B82F6;
  --c-health: #00C9A7;
  --c-study:  #9B59B6;
  --c-home:   #F97316;
}
```

### Шрифты

Подключить в `base.html` через Google Fonts CDN:

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link href="https://fonts.googleapis.com/css2?family=Unbounded:wght@700;800&family=Onest:wght@400;500;600;700&display=swap" rel="stylesheet">
```

- `font-family: 'Onest', sans-serif` — основной шрифт `body`
- `font-family: 'Unbounded', sans-serif` — логотип, заголовки страниц (`.page-title`), hero h1, заголовки форм

### Иконки

Font Awesome 6.5.0 через CDN (уже есть в проекте, сохрани):
```html
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.5.0/css/all.min.css">
```

## Навбар (base.html)

Навбар одинаков на всех страницах. Реализовать в `base.html`.

**Структура:**
- Тёмный фон `var(--navy)`, высота 58px, sticky top
- Слева: логотип HabitFlow (Unbounded, 14px) с градиентным квадратом-иконкой
- Центр/flex: навигационные ссылки с иконками FA:
  - Задачи → `fa-regular fa-circle-check`
  - Привычки → `fa-solid fa-arrow-trend-up`
  - Статистика → `fa-solid fa-chart-bar`
- **Кнопку «Главная» не добавлять** — логотип кликабелен и ведёт на `/`
- Справа (авторизованный): аватар-кружок с первой буквой email + email + шеврон (дропдаун с «Выход»)
- Справа (гость): кнопка «Вход» в стиле `rgba(108,99,255,.25)` с рамкой
- Активная ссылка: `background: rgba(108,99,255,.35)`, цвет белый

**Важно:** Jinja2-условие для гостя/пользователя уже существует в проекте — сохрани его логику, замени только HTML/CSS.

## Мобильная адаптация навбара

Добавить в `design-system.css` breakpoint `@media (max-width: 768px)`:

- Скрыть текстовые подписи в навбаре, оставить только иконки FA (добавить класс `.nav-link-text` на текст и скрывать его)
- Логотип: скрыть текст «HabitFlow», оставить только иконку-квадрат
- Email пользователя: скрыть, оставить только аватар-кружок
- Навбар не должен переполняться и уходить за край экрана

## Страницы и шаблоны

### 1. Главная — `src/templates/index.html`

**Layout:** двухколоночный — sticky sidebar 260px слева + основной контент справа.

**Sidebar (sticky):**
- Блок «Темы»: список тем пользователя как radio-фильтр. Каждая тема — цветная точка + название. Активная: `background: #EEF0FF`, цвет `var(--accent)`.
- Блок «Статистика»: 5 метрик в серых плашках. Последняя — прогресс-бар «Успех сегодня» с семантическим цветом: 0–33% → `var(--danger)`, 34–66% → `var(--warn)`, 67–100% → `var(--mint)`.

**Hero-баннер:** тёмно-фиолетовый градиент, Unbounded заголовок, два badge справа (дата + количество привычек на сегодня).

**Сетка 2 колонки:**
- Карточка «Задачи»: последние 5 задач. Строка задачи = кликабельная область (ведёт на редактирование), цветная полоска слева = цвет темы, бейдж справа = цвет темы. **Без чекбоксов.**
- Карточка «Привычки на сегодня»: до 4 привычек. Прогресс-бар, кнопки «Изменить» и «Выполнить».

**Цитата:** блок с жёлтым фоном внизу.

### 2. Задачи — `src/templates/tasks/list.html`

**Структура страницы:** полная ширина, max-width 900px, центрировано.

**Фильтры:**
- Пилюли тем (горизонтальный ряд) — клик меняет GET-параметр `theme_id`
- Белая полоска с тремя select (Статус, Сортировка) и toggle Возрастание/Убывание — сабмит формой GET или JS

**Список задач:** каждая задача — карточка (`border-radius: 12px`) с:
- Цветной полоской слева (4px) = цвет темы
- Название + описание
- Справа: бейдж даты, бейдж темы, бейдж приоритета (низкий/средний/высокий с цветовой семантикой)
- Иконочные кнопки редактирования и удаления (без текста)
- Hover: лёгкая тень и смена border-color

**Кнопка «Добавить задачу»** в правом верхнем углу, градиентная.

**Пагинация** внизу.

### 3. Привычки — `src/templates/habits/list.html`

Аналогично задачам по структуре, но карточка привычки включает:

Центрированная карточка (max-width 440px) на фоне `var(--bg)` с тонким градиентным пятном.

**Вход:**
- Бейдж «Безопасный вход» (фиолетовый)

## Правила имплементации

1. **Не трогать backend-логику** — только шаблоны и статику.
2. **Сохранить все существующие Jinja2-переменные** (`{{ user }}`, `{{ tasks }}`, `{% for %}` и т.д.) — только обернуть в новую HTML-структуру.
3. **Сохранить все существующие `action=""` у форм**, CSRF-токены, методы POST/GET.
4. **CSS выносить в отдельные файлы** по модулям: `design-system.css`, `navbar.css`, `tasks.css`, `habits.css`, `stats.css`, `themes.css`, `forms.css`, `auth.css`. Подключать через `base.html`.
5. **JS выносить в** `src/static/js/ui.js` — интерактив пилюль тем, toggle порядка, показ пароля, прогресс-бар статистики.
6. **Не использовать Bootstrap, jQuery или другие фреймворки** — только нативный CSS и vanilla JS.
7. **Мобильная адаптация** реализуется в отдельной задаче после проверки десктопной версии. На этом этапе добавить только breakpoint для навбара (см. выше).
8. При конфликте между прототипом и существующей Jinja2-логикой — сохранить логику, адаптировать визуал.
