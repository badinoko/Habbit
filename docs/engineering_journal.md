# Engineering Journal

## 1. Назначение
`engineering_journal.md` — короткий инженерный журнал по задачам.

Фиксирует:
- почему принято конкретное решение;
- какие альтернативы рассматривались;
- какие риски были найдены на code review;
- какие выводы полезно не потерять.

## 2. Формат записи

### Запись YYYY-MM-DD — <короткий заголовок>
- Контекст: что меняли и зачем.
- Решение: что реализовали.
- Альтернативы: что рассматривали и почему отвергли.
- Риски: что может сломаться.
- Проверка: какие тесты/проверки запускали.
- Результат: что в итоге получили.
- Следующий шаг: что делать дальше.

## 3. Принципы ведения
- Писать кратко и по фактам.
- Не дублировать diff кода (для этого есть git).
- Фокус на reasoning и lessons learned.
- Запись делается после завершения значимой задачи или нормализации (`N1`, `N2`, `N3`).

## 4. Шаблон для копирования
