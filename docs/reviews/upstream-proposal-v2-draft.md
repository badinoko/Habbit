# Предложение Для Upstream: Форк HabitFlow — Версии v2 и v3

Этот документ собирает отличия между публичной базой `v1.0` и последующими версиями в нашем форке. Он нужен как накопительный handoff для передачи оригинальному автору.

Три контрольных точки:

- тег `v1.0` и опубликованный сайт `https://www.amir-vps.ru/` отражают начальный UI-срез;
- тег `v2.0` — shell-redesign, form reliability, stats expansion, security hardening;
- тег `v3.0` — visual identity pass: hero-баннеры, карточные акценты, цветовой пикер.

## База Сравнения

- upstream repository: `https://github.com/Qwertyil/HabitFlow`
- наш лабораторный форк: `https://github.com/badinoko/Habbit`
- ранняя контрольная точка: `v1.0`
- текущая рабочая ветка накопления v2: `main`

## Что Уже Добавлено В v2 Поверх v1.0

### Визуальная система и shell

- shell переведён на более спокойный sans/display split: рабочая типографика в `Onest`, display-role в `Cormorant Garamond`;
- navbar стал плотнее, контрастнее и менее “серым” по восприятию;
- user dropdown упрощён до одного действия `Выход` вместо перегруженного меню;
- theme toggle в shell сделан визуально активным, а не монохромно-приглушённым;
- для static assets добавлен cache-busting query version, чтобы браузер не держал старые CSS/JS после деплоя.

### Главная, темы и page density

- homepage shell выровнен по вертикальным offset'ам без стартового микроскролла;
- блок `Мои темы` и пустое состояние пересобраны так, чтобы не было одновременно add-tile и сообщения “Темы не найдены”;
- форма `Создание темы` получила более компактный desktop-header без лишней высоты и без ненужного микроскролла на Full HD.
- длинные названия тем и пользовательских лейблов больше не раздувают sidebar, theme cards и user-trigger в navbar: они режутся по ellipsis, а полное значение доступно через tooltip.
- в form theme chooser для задач и привычек длинные пользовательские имена больше не режутся преждевременно и не вылезают из карточки: selector перешёл на более широкие responsive cards с переносом внутри самой карточки и естественным переходом в новые ряды.

### Статистика

- отдельный дублирующий header на `/stats` убран;
- вместо длинного вертикального сценария добавлен компактный sticky toolbar с переключением секций;
- секции `Обзор / Задачи / Привычки / Темы / Инсайты` реально переключают видимую панель, а не просто дёргают страницу;
- range switcher сохраняет текущую секцию при переключении периода;
- sticky offset toolbar выровнен с shell, чтобы не было стартового “подползания” при первом скролле.

### Form UX и client-side надёжность

- submit-логика формы собрана в единый pipeline вместо конкурирующих submit-обработчиков;
- client-side ошибки теперь показываются рядом с блоком действий формы, а не только в самом начале длинного form layout;
- кастомные ошибки привычек дополнительно подсвечивают связанный блок/поле inline;
- ошибки диапазона дат в форме привычки теперь таргетятся в `ends_on`, даже если выбран weekly schedule: интерфейс прокручивает и подсвечивает именно проблемное поле, а не unrelated block с днями недели;
- loading-state теперь включается только после клиентских проверок и снимается при ошибке;
- PUT-формы больше не отправляют запрос, если локальная валидация уже отменила submit.
- оставшиеся browser-native delete confirms заменены на одну общую in-app modal confirmation без layout-shift при открытии.

### Content safety и payload hygiene

- screenshots из старого `v1.0` показали, что в системе могли храниться script-like payloads вида `<script>alert(...)</script>` и мусорные строки с плохим переносом;
- по наблюдаемому поведению это выглядело как escaped text, а не как реально выполнившийся JavaScript: payload был напечатан на экране, но не исполнялся как код;
- это всё равно плохое состояние продукта: даже без успешной XSS такие payloads засоряют UI и ломают восприятие страниц;
- клиентские notifications больше не собираются через `innerHTML`: `ui.js` переведён на DOM API, чтобы даже служебные сообщения не открывали лишний XSS-риск при будущих изменениях;
- в v2 уже добавлены overflow guards для длинных строк, а отдельным следующим hardening-pass стоит удерживать такие имена на входе через stricter server-side validation against control chars and markup-like payloads.

## Что Исправляет Конкретно По Замечаниям К v1.0

- убран сценарий “нажал создать и ничего не произошло”, когда ошибка была слишком далеко от кнопки;
- убрано залипание submit-кнопки после клиентской ошибки;
- убран скрытый дефект, при котором edit-form могла отправить `PUT` несмотря на локальную ошибку периода;
- убрана зависимость поведения формы от случайного порядка submit-listeners.

## Что При Этом Не Менялось

- проект остаётся server-rendered приложением, без перехода в SPA;
- scope не расширялся до команд, админки, reminders или due dates;
- доменная модель задач, привычек и тем не перепридумывалась ради v2;
- основная идея продукта остаётся в персональном task/habit workflow, а не в долгосрочном planning-suite.

## Что Добавлено В v3 Поверх v2

Версия 3.0 — это visual identity pass. Если v2 чинила поведение и перестраивала shell, то v3 переносит визуальный характер с главной страницы на все остальные экраны. До v3 внутренние страницы выглядели как таблицы: чёрный текст на белом фоне, шрифт Onest везде, плоские карточки.

### Градиентные hero-баннеры на всех страницах

- `/tasks`, `/habits`, `/themes` получили полноширинные hero-секции с градиентными фонами, крупным заголовком в Cormorant Garamond и описанием;
- каждая страница имеет свой цветовой градиент: задачи — navy→purple, привычки — teal→mint, темы — purple→orange;
- `/stats` получила свой hero-баннер с indigo→blue градиентом;
- формы создания/редактирования (`/tasks/new`, `/habits/new`, `/themes/new` и их edit-версии) получили компактные hero-заголовки с иконкой и градиентом, соответствующим разделу;
- eyebrow-бейджики («Раздел задач», «Трекинг привычек» и т.д.) убраны после отзыва как дублирующие заголовок;
- высоты hero-баннеров унифицированы между всеми страницами.

### Акцентированные карточки задач и привычек

- карточки задач получили 5px боковую полоску цвета темы с цветной тенью и мягкий градиентный фон через `color-mix()`;
- карточки привычек получили аналогичную обработку: расширенная акцентная полоска и hover-эффект с усилением цвета темы;
- hover-состояние карточек включает `box-shadow` и увеличение непрозрачности цвета в градиенте.

### Per-section акценты на странице статистики

- каждая секция `/stats` (Задачи, Привычки, Темы, Инсайты) получила цветной `border-top` и subtile radial-gradient фон в цвете секции;
- задачи — фиолетовый акцент, привычки — mint, темы — оранжевый, инсайты — синий.

### Переработанный цветовой пикер для тем

- старые карточки с hex-кодами (#E74C3C) заменены крупными круглыми свотчами 48×48px;
- выбранный цвет подсвечивается двойным кольцом с галочкой;
- опция «Случайный» отображается радужным conic-gradient с иконкой кубика;
- палитра расширена с 8 до 11 цветов (добавлены розовый, циан, лайм), 12 элементов помещаются в одну линию на десктопе;
- обновлены обе формы: создание и редактирование темы.

### Типографика

- Cormorant Garamond применён для заголовков hero-баннеров на всех страницах (ранее только на главной);
- Onest сохранён для UI-контролов, body-текста, навигации — как и было задумано в v2-direction.

### Главная страница

- eyebrow-бейдж «Ежедневный дашборд» убран для единообразия с остальными страницами.

### Тёмная тема

- все hero-баннеры, карточные акценты и цветовой пикер корректно работают в тёмном режиме без дополнительной настройки: градиенты самодостаточны, не зависят от theme-токенов.

## Что При Этом Не Менялось (v2 → v3)

- серверная часть не менялась, кроме расширения палитры цветов тем (добавлено 3 новых цвета);
- проект остаётся server-rendered, без SPA-перехода;
- scope не расширялся — только визуальная идентичность;
- доменная модель, API, тесты и security-слой не затрагивались.

## Маршрут Просмотра Для Автора

1. **Главная** `/` — hero-баннер без бейджа, Cormorant Garamond заголовок, цитата
2. **Задачи** `/tasks` — navy→purple hero, карточки с цветными акцентами
3. **Привычки** `/habits` — teal→mint hero, карточки с тематическими градиентами
4. **Темы** `/themes` — purple→orange hero
5. **Создание темы** `/themes/new` — новый цветовой пикер с круглыми свотчами
6. **Статистика** `/stats` — indigo→blue hero, per-section цветные рамки
7. Переключить тёмную тему — все герои и пикер работают корректно
